Exploit Explorer (PT EE)

утилита которая позволяет оперативно искать ссылки на эксплоиты, в базе данных Rapid7 и exploit-db, по результатам сканирования сетевой инфраструктуры.

 

Текущая версия программы 2.4.3 build 11

Скачать последнюю версию.

Возможности:
  • Обработка файлов отчётов генерируемых МР8 в формате xml, mht, pdf;
  • Обработка текстовых файлов содержащих список CVE уязвимостей составленный вручную в произвольном формате
  • Обработка файлов отчётов других систем обнаружения уязвимостей в любых не сжатых форматах;
  • Поиск актуальных эксплоитов в общедоступных базах данных;
  • Генерация отчётов в формате html, csv, txt.
Требования:
  • JVM 1.7 или старше;
  • Свободной оперативной памяти не менее 512Мб;
  • доступ в интернет.
Применение:
  • В МР8 по результатам сканирования сформировать отчёт об уязвимостях в формате xml;
  • Сохранить отчёт в папку с утилитой;
  • GUI режим работы:
    • Загрузить файл отчёта нажатием кнопки "Файл отчёта";
    • Для поиска эксплоитов нажать кнопку "Найти эксплоиты";
    • Выгрузить результаты поиска можно нажатием кнопки "Выгрузить результаты поиска";
    • Полученный файл с расширением html можно просматривать в любом браузере;
  • Консольный режим работы:
    • Запустить пакетный файл на исполнение в командной строке с именем отчёта в качества параметра, например: ptee.bat "vulnerability report.xml" (кавычки вокруг имени файла обязательны, если в имени файла содержаться пробелы);
    • Файл с расширением csv можно просматривать и редактировать в MS Excel или OpenOffice Calc;
    • Полученный файл с расширением html можно просматривать в любом браузере;
Настройки:
  • Ключ -csv результаты поиска будут сохранены в файл с расширением csv;
  • Ключ -html результаты поиска будут сохранены в файл с расширением html;
  • Ключ -txt результаты поиска будут сохранены в файл с расширением txt.

Все настройки прописаны в пакетном файле идущем в комплекте поставки и могут быть изменены по вашему усмотрению.

 

Пример запуска утилиты из командной строки:
java -jar mp8scare.jar -html -threads8 "vulnerability report.xml"

 

Пример запуска с использованием пакетного файла:
ptee.bat "vulnerability report.xml"

 

Пример пакетного файла ptee.bat
@echo off
rem для изменения объёма используемой памяти
rem необходимо изменить параметр -Xmx512M
rem на другой, например: -Xmx128M или -Xmx1G
 
java -Xmx512M -jar mp8Scare.jar -html -threads24 %1

 

Интерпретация результатов:

Каждому эксплоиту присвоен ранг на основе его потенциального влияния на целевую систему.

Значение ранга может принимать одно из следующих значений в порядке убывания надёжности:

  • Excellent - Эксплоит ни когда не приводит к краху сервиса. Особенно это касается SQL инъекций, CMD execution, RFI, LFI и т.д и т.п.;
  • Great       - Эксплоит использует цель по умолчанию или автоматически её определяет или использует для конкретного приложения обратный адрес после проверки версии атакуемого приложения;
  • Good        - Эксплоит использует цель по умолчанию в общем случае для этого типа атакуемого приложения;
  • Normal     - Эксплоит достаточно надёжный, но зависит от конкретной версии и может не достаточно надёжно произвести автоопределение атакуемого приложения;
  • Average   - Эксплоит как правило не надёжный или трудно его использовать;
  • Low          - Эксплоит практически не возможно использовать или не срабатывает более чем в 50% случаев;
  • Manual    - Эксплоит не стабилен или трудно его использовать(в основном для DoS атак), так же этот рейтиг используется когда эксплуатация модуля не возможна без специальной настройки пользователем.
